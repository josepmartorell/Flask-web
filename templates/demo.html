Next development technology: PyTorch

In the changing landscape of artificial intelligence development, choosing which technology to invest development hours in is crucial and involves choosing taking into account many factors, such as whether it is cross-platform or being a maintained open source technology by a growing community of developers. In fact, each new technology must be chosen in such a way that it will be absorbed by those that have already come into play and that have already been implemented successfully, in such a way that it complements the previous ones with its new functionalities. For me, a smart choice is Pytorch. In essence, PyTorch provides two main features: an n-dimensional tensor, similar to numpy but that can be run on GPUs, and automatic differentiation to build and train neural networks. The most fundamental concept in PyTorch is the tensor, which is conceptually identical to a numeric matrix only that it is an n-dimensional matrix. PyTorch provides many functions to operate on these tensioners. PyTorch has support to run on graphics cards (GPU), it uses internally CUDA, an API that connects the CPU with the GPU that has been developed by NVIDIA.
